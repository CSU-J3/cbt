## SKILL.md

- Hosted Turso runs a stateless planner with ANALYZE blocked, so it will NOT use a covering or partial index for these gated aggregates unless forced with an INDEXED BY hint. Always pair the index with the hint. Confirmed across getFeedStats, getCorpusStats, getStageDistribution, getTopicDistribution, getFeedBills (277-279).
- `idx_bills_summary_feed` (partial, `WHERE summary IS NOT NULL`) is the shared index for the summary-gated COUNT class: getFeedStats, getCorpusStats(true), and getFeedBills's COUNT all hint onto it (their default WHERE all carries `summary IS NOT NULL`). The gated stage/topic distributions use their own partials (`idx_bills_summary_stage`, `idx_bills_summary_topics`, both `WHERE summary IS NOT NULL`); getSponsorProductivity uses `idx_bills_sponsor_agg`.
- Conditional INDEXED BY hints must be gated to the exact path whose WHERE matches the partial index's predicate. A too-broad hint forces the wrong index on sibling paths: the /bills?stage= regression in 279, where a blanket `!cluster` hint pessimized the stage path (152ms natural to 8.5s) until it was narrowed to the bare gated case. Gate narrowly, and verify the filtered/sibling paths after adding a hint.
- getFeedStats is the masthead count and runs on every HeaderBar page, so its mis-plan was app-wide, not route-local. Fixing it in 277 protected every inner page at once.

## docs/oddities.md

- Gated-aggregate mis-plan class: adding a gating predicate (`summary IS NOT NULL`) to a fat-table COUNT/aggregate makes the stateless planner intermittently drop it onto a full bills scan or a multi-index OR, past the 10s abort. "Cold-start 500" was a misnomer; it's plan nondeterminism, slow even fully warm. Remedy: partial/covering index matching the predicate plus a gated INDEXED BY hint.
- Conditional-hint over-gating: gating an INDEXED BY hint too broadly forces the wrong index on sibling query paths (the /bills?stage= case in 279). Verify the filtered/sibling paths after adding any hint.
