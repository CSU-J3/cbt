# HO 132.1 ‚Äî Bubble Click ‚Üí Slide-Out Drawer

## Context

HO 132 just shipped two bubble charts on the dashboard. Clicking a bubble today navigates to `/feed?stage=<slug>` or `/feed?topics=<slug>` ‚Äî full page change.

OTX pattern is different: clicking a malware cluster doesn't navigate, it opens a side pane with the related items. CBT should match that interaction. Click a stage bubble ‚Üí drawer slides in from the right edge with bills in that stage. Click a topic bubble ‚Üí drawer updates to filter by stage AND topic (coupled). Close the drawer ‚Üí selection clears.

The dashboard layout from HO 132 stays exactly as it is. No row reshuffling. The drawer is the only addition.

## Behavior

- Default state: drawer hidden. Dashboard renders as today.
- Click stage bubble ‚Üí drawer slides in from right. Width ~440px on desktop, full-screen on mobile. URL gains `?stage=<slug>`.
- Click topic bubble (with stage already selected): drawer updates content to filter by stage AND topic. URL becomes `?stage=<a>&topic=<b>`. **Bubbles are coupled ‚Äî both filters apply together. No toggle.**
- Click a different bubble in the same dimension: replaces that dimension's filter. (Clicking COMMITTEE then FLOOR = filter is FLOOR, not both.)
- Re-click the currently-selected bubble: clears that dimension's filter. If both were set and one clears, drawer stays open with remaining filter. If both clear, drawer closes.
- Close drawer via: X button in drawer header, backdrop click, ESC key. Any close clears all filters.
- Cmd/Ctrl+click on a bubble: bypass drawer, open `/feed?stage=‚Ä¶` in new tab (preserves the keyboard power-user path).

## Drawer content

Header (sticky top):
- Active filter chips: `STAGE: COMMITTEE √ó TOPIC: HEALTH` (each chip has its own √ó to clear just that dimension)
- Total count: "1,287 bills"
- View all in feed link: `‚Üí /feed?stage=committee&topics=healthcare`
- Close button (X)

Body (scrollable):
- 10 most recent matching bills using the compact `BillRow` variant from HO 125
- "View all in feed ‚Üí" footer link at the bottom (duplicate of header link for convenience)

If the filter returns zero bills: show "No bills match this filter." with the active chips still visible so the user can clear and try again.

### Proposals to post

1. **URL state shape.** `?stage=<slug>&topic=<slug>` ‚Äî same param names as `/feed` for round-trip consistency (cmd-click ‚Üí `/feed` carries the same filter naturally). Note: `/feed` uses `?topics=` (plural). Pick: match `/feed`'s `topics` plural, or use singular `topic` on home and translate at the cmd-click handler. Recommend match `/feed` so cmd-click is literal pass-through.

2. **Drawer component shape.** Client island `<DashboardBillsDrawer />` reading URL params via `useSearchParams()`. Renders nothing when no params present; renders the drawer when either or both present. Handles ESC + backdrop click + X button.

3. **Animation.** Slide from right, 200ms ease-out on open, 150ms ease-in on close. Backdrop fades 0 ‚Üí 0.5 alpha in parallel. CSS transitions only, no JS animation library.

4. **Drawer width.** 440px desktop, 100vw on mobile (‚â§700px). Full viewport height. Position fixed top:0 right:0.

5. **Backdrop.** Semi-transparent black scrim over the dashboard at ~0.5 alpha, click-to-close. Z-index above all dashboard content.

6. **Bubble click handler change.** Intercept the click on `<a>` elements ‚Äî `e.preventDefault()` on plain click, `router.push('/?stage=‚Ä¶&topic=‚Ä¶', { scroll: false })`. Cmd/Ctrl-click bypasses preventDefault, browser handles the `<a href>` natively (opens new tab).

7. **Couple logic.** On stage bubble click: `?stage=<new>` (replaces stage, preserves topic if set). On topic bubble click: `?topics=<new>` (replaces topic, preserves stage). Drawer queries with whatever combo of params is present.

8. **Selected bubble visual state.** Add `selected` class on the active bubble. Stroke `--accent-amber` thicker (same as HO 128 `.selected` style).

## Phase 2 ‚Äî Implementation (after sign-off)

### Step 1 ‚Äî Query + drawer component (commit A)

- Extend or add `getBillsByStageAndTopic({ stage?, topic? }, limit)` in `lib/queries.ts`. Handles four cases: both, stage only, topic only, neither (returns empty). Cache with `unstable_cache`, tag `bills`.
- Build `components/DashboardBillsDrawer.tsx`. Client island. Reads `useSearchParams()` for `stage` and `topics`. Renders backdrop + drawer panel when either param set.
- Header per spec (filter chips with per-dimension clear, total count, view-all link, X close).
- Body: server-rendered bills via a server component child fed the filter. Compact BillRow rows.
- Empty state per spec.
- Dismiss: X, backdrop click, ESC handler in useEffect.

### Step 2 ‚Äî Bubble click rewiring (commit B)

- Modify `DashboardBubbleChart.tsx` click handling:
  - `<a href>` stays for cmd-click (passes through to /feed natively)
  - Default click: `e.preventDefault()`, then `router.push` with the new URL params, merging with existing params (so clicking topic preserves stage and vice versa).
  - Re-click on selected bubble: remove that param from URL.
- Add `selected` visual state on the active bubble in each chart (read URL params, compare to bubble id).

### Step 3 ‚Äî Mount + verify (commit C)

- Mount `<DashboardBillsDrawer />` in `app/page.tsx` once, alongside the existing grid.
- Confirm SSR/CSR boundary: drawer is a client island, bills inside it can still be server-rendered children.

## Don't

- Don't reshuffle the dashboard grid. Row 1 + Row 2 stay as HO 132 left them.
- Don't add the drawer to any page other than `/`.
- Don't introduce a third-party drawer/modal library. CSS transitions + a small client component is all this needs.
- Don't make the drawer scroll-lock the body for now. Backdrop click-to-close is the dismiss path; preventing background scroll is a polish item if it bothers in practice.
- Don't bundle dashboard click handler changes with the existing HO 134 Phase 3 nav work. These are independent shipments.
